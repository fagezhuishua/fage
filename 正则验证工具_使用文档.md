# 正则验证工具（regex.py）使用文档

> 版本：v1.1 Settings Edition  
> 技术栈：Python 3 + customtkinter + re

---

## 一、功能概述

`regex.py` 是一款基于 modern GUI（customtkinter）的 **正则表达式实时验证工具**，主要用于快速测试、验证和调试正则表达式，适合开发、运维、安全分析人员使用。

### 核心功能

| 功能 | 说明 |
|------|------|
| 预设模板 | 内置 IPv4、IPv6、域名、URL、邮箱、手机号、身份证、UA、MAC、端口等常用正则 |
| 实时匹配 | 输入正则或测试内容时自动触发匹配（KeyRelease 事件） |
| 编译错误提示 | 正则语法错误时显示详细错误信息和位置 |
| 匹配高亮 | 匹配行绿色高亮，未匹配行红色高亮，捕获组单独展示 |
| 标志位配置 | 支持忽略大小写、多行模式、`.` 匹配换行 |
| 多语言 | 支持中文 / 英文界面 |
| 主题切换 | 支持浅色 / 深色模式 |
| 个性化设置 | 支持字体大小、字体颜色、边框颜色、边框粗细自定义 |

---

## 二、运行环境

### 2.1 必需依赖

```bash
pip install customtkinter
```

### 2.2 运行方式

直接双击运行或通过命令行启动：

```bash
python regex.py
```

> 注：该程序为 GUI 应用，**没有命令行参数**。

## 三、界面与操作说明

启动后界面分为 **左侧边栏** 和 **右侧主工作区**。

### 3.1 左侧边栏

| 控件 | 功能 |
|------|------|
| **预设模板** | 快速选择内置正则模板 |
| **运行匹配** | 手动触发一次匹配验证 |
| **⚙️ 设置** | 打开设置窗口，自定义界面外观 |
| **粘贴示例** | 自动填入手机号验证示例 |
| **清空全部** | 清空正则、测试内容、结果区域 |
| **深色模式** | 切换浅色 / 深色主题 |

### 3.2 正则输入区域

- 顶部输入框输入正则表达式。
- 右侧复选框配置 `re` 标志位：
  - **忽略大小写**（`re.IGNORECASE`）
  - **多行模式**（`re.MULTILINE`）
  - **`.` 匹配换行**（`re.DOTALL`）
- 支持从预设模板自动填充。

### 3.3 测试内容区域

- 粘贴待验证的文本内容，**每行一条**。
- 点击右上角 `✕` 可清空内容。
- 默认显示占位提示文本。

### 3.4 结果区域

- 显示逐行匹配结果。
- 顶部显示统计信息：`匹配: X / 未匹配: Y / 总计: Z`。
- 颜色含义：
  - **绿色背景**：匹配成功
  - **红色背景**：未匹配
  - 捕获组会在匹配行下方以 `└─ 捕获组: ...` 形式展示

## 四、参数与配置说明

### 4.1 程序配置文件

配置文件名：`regex_tool_settings.json`  
位置：与 `regex.py` 同目录  
编码：UTF-8

#### 默认配置项

```json
{
  "language": "zh_CN",
  "font_size": 13,
  "font_color_light": "#000000",
  "font_color_dark": "#ffffff",
  "border_color_light": "#cccccc",
  "border_color_dark": "#333333",
  "border_width": 1
}
```

| 参数 | 类型 | 可选值 | 说明 |
|------|------|--------|------|
| `language` | 字符串 | `zh_CN` / `en` | 界面语言，修改后需重启生效 |
| `font_size` | 整数 | 10 ~ 20 | 输入/输出区字体大小 |
| `font_color_light` | 颜色值 | 十六进制色值 | 浅色模式字体颜色 |
| `font_color_dark` | 颜色值 | 十六进制色值 | 深色模式字体颜色 |
| `border_color_light` | 颜色值 | 十六进制色值 | 浅色模式边框颜色 |
| `border_color_dark` | 颜色值 | 十六进制色值 | 深色模式边框颜色 |
| `border_width` | 整数 | 0 ~ 3 | 文本框边框粗细 |

### 4.2 内置预设模板

| 模板名称 | 正则说明 |
|----------|----------|
| `Custom / 自定义` | 空模板，自行输入 |
| `IPv4 Address` | 标准 IPv4 地址 |
| `IPv6 Address` | 简单 IPv6 地址格式 |
| `Domain Name` | 域名格式 |
| `URL` | HTTP/HTTPS URL |
| `Email` | 电子邮箱 |
| `China Mobile` | 中国大陆手机号 |
| `China ID Card` | 中国大陆身份证号 |
| `User-Agent` | 常见 User-Agent 头 |
| `MAC Address` | MAC 地址 |
| `Port Number` | 合法端口号（1-65535） |

### 4.3 正则标志位

| 标志位 | Python 常量 | 说明 |
|--------|-------------|------|
| 忽略大小写 | `re.IGNORECASE` | 匹配时不区分大小写 |
| 多行模式 | `re.MULTILINE` | `^` 和 `$` 匹配每行开头/结尾 |
| `.` 匹配换行 | `re.DOTALL` | `.` 可以匹配换行符 `\n` |

## 五、匹配规则说明

### 5.1 匹配逻辑

程序使用 `re.compile(pattern, flags)` 编译正则，然后对测试内容的每一行调用 `regex.search(line)`：

- 只要行内**任意位置匹配**，即视为匹配成功。
- 如需整行匹配，请在正则中使用 `^` 和 `$`。
- 捕获组（`()`）会被提取并在结果中展示。

### 5.2 结果展示示例

测试内容：

```
13800138000
12345678901
15088886666
```

正则：`^1[3-9]\d{9}$`

结果：

```
✅ 13800138000
❌ 12345678901   ⬅ 未匹配
✅ 15088886666
```

### 5.3 捕获组展示示例

正则：`^(\d{3})-(\d{4})-(\d{4})$`

测试内容：`010-1234-5678`

结果：

```
✅ 010-1234-5678
   └─ 捕获组: 010 | 1234 | 5678
```

## 六、注意事项

### 6.1 实时匹配性能

- 程序通过 `<KeyRelease>` 事件触发实时匹配。
- 当测试内容**非常大**（如数千行）或正则**非常复杂**时，频繁输入可能导致界面卡顿。
- 建议大数据量测试时，先粘贴内容再输入正则，或使用较简单的正则。

### 6.2 空行处理

- 测试内容中的空行会被保留为换行，不参与匹配统计。
- 仅包含空白字符的行也按空行处理。

### 6.3 正则编译错误

- 正则语法错误时，结果区会显示红色错误信息，并尝试指出错误位置。
- 常见错误：括号不匹配、转义错误、非法量词等。

### 6.4 预设模板限制

- 内置模板为常用简化版，不一定覆盖所有边缘场景。
- 例如 IPv6 模板为简化格式，复杂的零压缩写法可能无法完全匹配。
- 生产环境使用前，请根据实际需求验证正则的准确性。

### 6.5 设置生效规则

- 字体大小、颜色、边框等设置**保存后立即生效**。
- **语言设置修改后需重启程序**才能生效。
- 设置保存到同目录的 `regex_tool_settings.json` 文件中。

### 6.6 结果区状态

- 结果区默认只读，不可编辑。
- 如需复制结果，可直接选中文本后按 `Ctrl+C`。

### 6.7 数据安全

- 程序仅在本地运行，不会上传任何数据。
- 配置文件仅保存外观设置，不保存正则或测试内容。

## 七、常见问题（FAQ）

**Q1：为什么输入正则后没有实时匹配？**

请检查测试内容区域是否已填入内容，且内容不是默认占位提示文本。点击输入区获取焦点后占位符会自动消失。

**Q2：为什么我的正则匹配不到内容？**

- 检查是否开启了正确的标志位（如忽略大小写、多行模式）。
- 检查正则是否使用了 `^` 和 `$` 进行整行匹配。
- 注意本工具按行调用 `search()`，不是 `match()`，因此不需要从行首开始匹配。

**Q3：如何测试多行内容？**

直接将多行文本粘贴到测试内容区域即可。若需要使用 `^`/`$` 匹配每行，请勾选「多行模式」。

**Q4：为什么切换语言后界面没有变化？**

语言设置需要重启程序才能生效。

**Q5：如何添加自己的常用正则模板？**

当前版本不支持在界面中添加自定义预设。如需扩展，可编辑源码中的 `PRESETS` 字典，添加名称和正则即可。
