# 抓包分析工具使用文档

> **版本**：v2.0 Modern Edition  
> **适用系统**：Windows 7/10/11（64 位推荐）  
> **依赖环境**：Python 3.x + scapy + customtkinter（已打包为 exe 的可直接使用）  
> **支持格式**：`.pcap` / `.pcapng` / `.cap` / `.dmp`

---

## 一、功能模块

### 1. 文件导入
- **点击选择文件**：通过侧边栏或欢迎页的「选择文件」按钮打开文件对话框。
- **拖拽导入**：将抓包文件直接拖入窗口中央区域即可自动加载（Windows 原生拖拽）。
- **格式支持**：`.pcap`、`.pcapng`、`.cap`、`.dmp`。

### 2. 深度分析引擎

| 模块 | 说明 |
|------|------|
| **文件概览** | 显示文件名、路径、大小、包数量、捕获起止时间、持续时间、平均包长及 10 秒粒度流量时间线。 |
| **协议统计** | 统计 TCP / UDP / HTTP / TLS / DNS / ICMP / ARP / Other 等协议的包数和占比，识别常见服务（SSH/HTTP/HTTPS/DNS 等）。 |
| **TCP 连接分析** | 识别 TCP 三次握手、半开连接、RST 拒绝连接、所有 RST 包，并计算 SYN→SYN-ACK、握手总延迟。 |
| **TLS/证书分析** | 解析 ClientHello / ServerHello / Certificate，提取 SNI、TLS 版本、密码套件，检测弱 TLS/SSL 版本和 TLS Alert。 |
| **错误与异常** | 汇总 TCP RST、重传、重复 ACK、零窗口、ICMP 错误、HTTP 4xx/5xx、DNS 解析异常、TLS Alert、半开连接等风险项。 |
| **通信详情** | 展示 Top10 源 IP、目的 IP、通信对、目的端口，以及 TCP/UDP 会话数和独立 IP 数量。 |
| **综合结论** | 自动生成结构化分析报告，包含风险等级、需求匹配结果、针对用户问题的排查线索。 |

### 3. 分析需求配置
- **重点分析 IP**：配置关注的 IP 地址，分析结果中会用 `★` 高亮匹配项。
- **关注错误类型**：配置关注的错误关键字（如 `RST`、`重传`、`TLS`），匹配的错误会高亮显示。
- **具体问题描述**：输入业务问题（如「连接中断」「访问慢」「证书错误」），结论页会根据关键词给出排查线索。

### 4. 结果交互
- **标签页切换**：7 个结果标签页，可查看不同维度分析结果。
- **搜索高亮**：在当前标签页内搜索关键字并高亮显示，支持回车触发。
- **清除高亮**：一键清除所有标签页的搜索高亮。
- **导出 .txt**：将当前标签页内容导出为文本文件。
- **复制结果**：将当前标签页内容复制到剪贴板。
- **清空结果**：清空分析结果并回到欢迎页。

### 5. 设置与个性化
- **界面语言**：支持简体中文（`zh_CN`）和英文（`en`），切换后需重启生效。
- **外观主题**：支持深色 / 浅色模式切换。
- **字体大小**：10 ~ 20 可调。
- **字体颜色**：分别设置浅色/深色模式下的字体颜色。
- **边框样式**：分别设置浅色/深色模式下的边框颜色和边框粗细。
- **设置持久化**：配置自动保存到同目录下的 `packet_settings.json`。

## 二、操作流程

### 方式一：快速分析

1. **启动程序**：双击 `抓包分析工具.exe`。
2. **导入文件**：
   - 点击欢迎页或侧边栏的「📁 选择文件」按钮；或
   - 直接将 `.pcap` / `.pcapng` 文件拖入窗口。
3. **等待分析**：程序会显示进度条，自动完成加载与深度分析。
4. **查看结果**：分析完成后进入结果页，通过 7 个标签页查看文件概览、协议统计、TCP 连接、TLS、错误、通信详情和综合结论。
5. **导出/复制**：在结果页使用「导出 .txt」或「复制结果」保存所需内容。

### 方式二：针对性分析

1. **打开分析需求配置**：点击侧边栏「⚙️ 分析需求」。
2. **填写配置项**：
   - 重点分析 IP：输入关注的 IP，多个用英文逗号分隔。
   - 关注错误类型：输入关键字，如 `RST, 重传, TLS`。
   - 具体问题描述：描述你要排查的问题，如「访问网站慢」「连接经常断开」「TLS 证书报错」。
3. **保存配置**：点击「保存」。
4. **导入并分析抓包文件**：结果中匹配 IP 和错误类型的记录会显示 `★` 标记，结论页会针对你的问题给出排查建议。

### 方式三：重新分析

- 分析完成后，侧边栏的「🔄 重新分析」按钮会变为可用状态。
- 点击即可使用当前文件和最新配置重新分析。

## 三、注意事项

### 1. 运行环境
- 本工具基于 `scapy` 解析抓包文件，**首次使用源码运行前需确保已安装依赖**：
  ```bash
  pip install scapy customtkinter pyinstaller
  ```
- 已打包的 `.exe` 文件可在未安装 Python 的 Windows 电脑上运行。

### 2. 依赖与兼容性
- `scapy` 在 64 位 Windows 下存在已知的 `ctypes OverflowError` 提示，程序已做延迟导入处理，不影响使用。
- TLS/SSL 解析依赖 `scapy.layers.tls`，若 scapy 未完整安装 TLS 模块，TLS 分析会降级为仅检测 TLS 端口流量。
- 拖拽功能仅支持 Windows 平台（使用 Windows 原生 `WM_DROPFILES`）。

### 3. 文件与性能
- 大抓包文件（数百 MB 或数 GB）加载和分析可能需要一定时间，请耐心等待进度条完成。
- 分析过程中请勿关闭程序或反复点击按钮，以免线程异常。
- 程序会在同目录下生成 `packet_settings.json` 用于保存设置，删除该文件会恢复默认设置。

### 4. 结果解读
- **三次握手延迟**：`SYN → SYN-ACK` 时间过长通常表示网络链路或目标主机响应慢。
- **半开连接**：仅发出 SYN 未收到 SYN-ACK，可能是目标不可达、防火墙过滤或丢包。
- **RST 包**：连接被强制终止，可能是端口未开放、防火墙拒绝或应用层主动断开。
- **重传 / 重复 ACK**：通常意味着网络中存在丢包或乱序。
- **TLS Alert / 弱版本**：提示 TLS 握手失败、证书问题或使用了不安全的协议版本。
- **需求匹配高亮（★）**：仅表示该项命中了你配置的关注 IP 或错误关键字，不代表风险等级。

### 5. 常见问题

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 无法启动，提示缺少 scapy | 源码运行环境未安装依赖 | 执行 `pip install scapy customtkinter` |
| 拖拽文件无反应 | 非 Windows 系统或权限限制 | 使用「选择文件」按钮导入 |
| 分析大文件卡顿 | 文件过大或内存不足 | 拆分抓包文件或增加系统内存 |
| TLS 信息为空 | scapy TLS 模块未完整安装 | 重新安装/升级 scapy：`pip install -U scapy` |
| 导出的 txt 乱码 | 使用非 UTF-8 编码打开 | 用记事本/VS Code 选择 UTF-8 编码打开 |
