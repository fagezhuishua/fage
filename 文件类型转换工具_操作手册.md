# 文件类型转换工具（transfer.py）操作手册

> 版本：v3.1 i18n Edition  
> 技术栈：Python 3 + customtkinter + tkinterdnd2 + 多线程/COM

---

## 一、功能模块介绍

`transfer.py` 是一款基于 modern GUI 的文件类型转换工具，支持文档、表格、演示、图像、音频、数据格式等多种文件类型的批量互转。

### 1.1 系统架构

```
┌─────────────────────────────────────────┐
│              GUI 界面层                  │
│  （customtkinter + tkinterdnd2 拖拽）     │
├─────────────────────────────────────────┤
│              业务控制层                  │
│  （文件导入、格式校验、任务调度、UI 刷新） │
├─────────────────────────────────────────┤
│              格式转换引擎                │
│  （COM/Office、python-docx、pdf2docx、    │
│   pandas、Pillow、pydub 等）             │
└─────────────────────────────────────────┘
```

### 1.2 核心功能模块

| 模块 | 说明 |
|------|------|
| **文件导入模块** | 支持批量添加文件、添加文件夹、拖拽文件/文件夹到窗口 |
| **格式映射模块** | 维护源格式与目标格式的转换关系（`CONVERSION_MAP`） |
| **转换引擎模块** | 根据文件类型调用不同库执行转换 |
| **任务调度模块** | 多线程处理非 COM 文件，单线程处理 COM 文件 |
| **UI 刷新模块** | 批量刷新结果，限制结果区行数防止内存无限增长 |
| **设置模块** | 支持语言切换、主题切换、格式帮助查看 |

## 三、运行环境

### 3.1 必需依赖

```bash
pip install customtkinter
```

### 3.2 可选依赖

| 依赖 | 用途 | 缺少时的影响 |
|------|------|-------------|
| `tkinterdnd2` | 拖拽文件/文件夹到窗口 | 拖拽功能不可用，但按钮导入仍可用 |
| `python-docx` | 读取/生成 docx 文件 | txt/md/html → docx、docx → txt 不可用 |
| `pandas` | 表格格式转换 | xls/xlsx/csv 互转不可用 |
| `openpyxl` / `xlwt` / `xlrd` | Excel 读写 | pandas 读取/保存 Excel 失败 |
| `PyPDF2` | PDF 提取文本 | pdf → txt 不可用 |
| `reportlab` | 文本生成 PDF | txt → pdf 备用方案不可用 |
| `pdf2docx` | PDF 转 Word | pdf → docx 不可用 |
| `docx2pdf` | Word 转 PDF | docx → pdf 备用方案不可用 |
| `Pillow` | 图像格式转换 | 所有图像转换不可用 |
| `pydub` | 音频格式转换 | 所有音频转换不可用 |
| `pywin32` | 调用 Office COM | doc/xls/ppt 及其部分转换不可用 |
| `cairosvg` | SVG 矢量图转换 | svg → png/jpg/bmp 不可用 |
| `numpy` | 背景渐变加速 | 背景渲染稍慢，但不影响功能 |

### 3.3 Office 依赖说明

- **doc / xls / ppt 格式**：必须安装 Microsoft Office（Word/Excel/PowerPoint），并通过 `pywin32` 调用 COM 接口。
- **docx / xlsx / pptx 格式**：部分转换可不依赖 Office（如 docx → txt 用 python-docx），但 docx → doc/pdf 等仍需要 Office。

### 3.4 运行方式

```bash
python transfer.py
```

## 四、界面与操作说明

### 4.1 主界面布局

启动后界面分为 **左侧边栏** 和 **右侧主工作区**。

#### 左侧边栏

| 控件 | 功能 |
|------|------|
| **外观主题 / 深色模式** | 切换浅色 / 深色主题 |
| **语言** | 切换中文 / 英文 / 日文 / 德文（需重启生效） |
| **格式说明** | 打开支持格式与转换关系说明窗口 |
| **填入示例** | 生成示例文件（txt、docx、csv、png）并添加到列表 |

#### 右侧主工作区

| 区域 | 说明 |
|------|------|
| **源文件区** | 显示待转换文件列表，支持拖拽导入 |
| **配置行** | 选择目标格式、设置输出目录、开始转换 |
| **进度条** | 显示当前转换进度 |
| **转换结果区** | 显示转换成功/失败的详细信息 |

### 4.2 导入文件

支持三种导入方式：

1. **批量添加文件**：点击按钮，在文件对话框中多选文件。
2. **添加文件夹**：点击按钮选择文件夹，程序会自动递归扫描该文件夹下所有文件。
3. **拖拽导入**：直接从文件管理器拖拽文件或文件夹到程序窗口。

### 4.3 选择目标格式

- 在「目标格式」下拉框中选择输出格式。
- 程序会自动跳过不支持转换为该格式的源文件，并在结果区提示。

### 4.4 设置输出目录

- 默认输出目录：`~/Downloads/transfer_output`
- 点击「浏览...」可选择其他输出目录。
- 若目录不存在，程序会自动创建。

### 4.5 开始转换

- 点击「开始转换」按钮。
- 程序会先校验文件是否存在、格式是否支持。
- 转换过程中，「开始转换」按钮会禁用，进度条实时更新。
- 转换完成后会弹出成功提示框，可选择「打开目录」或「关闭」。

### 4.6 清空操作

- **清空输入**：清空源文件列表。
- **清空输出**：清空转换结果区显示的内容（不会删除已生成的文件）。

## 五、转换引擎说明

### 5.1 转换方式分类

| 文件类型 | 主要转换方式 |
|----------|-------------|
| 纯文本/标记语言 | Python 原生读写 + python-docx |
| docx | python-docx / docx2pdf / Office COM |
| doc / xls / ppt / rtf / odt | Office COM（必须安装 Office） |
| pdf | PyPDF2 / pdf2docx / reportlab |
| 表格 | pandas + openpyxl/xlwt |
| 图像 | Pillow |
| 音频 | pydub（底层可能依赖 ffmpeg） |
| 数据格式 | Python 标准库 json / csv / xml |

### 5.2 线程调度策略

- **非 COM 文件**：使用 `ThreadPoolExecutor` 多线程并发处理，最多 4 个线程。
- **COM 文件**（doc/xls/ppt 等）：单线程顺序处理，避免 COM 接口冲突。
- **文件夹扫描**：在后台线程执行，避免界面卡死。

### 5.3 临时文件处理

- 部分转换会先生成中间 docx 文件（如 txt → pdf），保存在系统临时目录。
- 转换完成后会自动清理临时文件。

## 六、注意事项

### 6.1 Office 相关转换

- 转换 `doc`、`xls`、`ppt`、`rtf`、`odt` 等格式时，**必须安装 Microsoft Office**。
- 转换过程中 Office 应用会在后台静默运行（Word/Excel/PowerPoint 不可见窗口）。
- 若 Office 未安装或 COM 接口异常，会提示「需要安装 Office」。
- 建议转换前关闭正在使用的 Office 文档，避免文件占用冲突。

### 6.2 依赖安装

- 使用前请根据目标格式安装对应依赖，否则会提示「缺少 xxx」。
- 推荐安装全部可选依赖以获得最佳体验：

```bash
pip install tkinterdnd2 python-docx pandas openpyxl xlwt xlrd PyPDF2 reportlab pdf2docx docx2pdf Pillow pydub pywin32 cairosvg numpy
```

### 6.3 输出文件覆盖

- 若输出目录已存在同名文件，程序会直接覆盖，请提前备份重要文件。

### 6.4 大文件与大批量转换

- 大文件（如大型 PDF、高分辨率图片）转换可能耗时较长，请耐心等待。
- 结果区最多保留 2000 行，超过会自动删除旧记录，防止内存无限增长。
- 图像转 ICO 时会自动生成 16×16 到 256×256 的多尺寸图标。

### 6.5 音频转换

- 音频转换依赖 `pydub`，部分格式（如 mp3）可能需要系统安装 `ffmpeg` 或 `libav`。
- 若音频转换失败，请检查是否已安装 ffmpeg 并添加到环境变量。

### 6.6 SVG 转换

- SVG 转位图需要安装 `cairosvg`，同时需要系统支持 cairo 图形库。
- Windows 用户可能需要额外安装 GTK+ / cairo 运行库。

### 6.7 PDF 转 Word

- `pdf → docx` 依赖 `pdf2docx`，对复杂排版 PDF 的转换效果可能不理想。
- 扫描版 PDF（图片型 PDF）无法直接转换为可编辑 docx。

### 6.8 语言设置

- 语言切换后需要**重启程序**才能生效。
- 语言配置保存在同目录的 `transfer_settings.json` 文件中。

### 6.9 拖拽导入限制

- 拖拽功能依赖 `tkinterdnd2`，若未安装则拖拽不可用。
- 拖拽大文件夹时，界面会显示「正在扫描文件夹...」，扫描完成后才显示文件列表。

### 6.10 权限问题

- 若源文件被其他程序占用，或无读取权限，会提示转换失败。
- 若输出目录无写入权限，会提示保存失败。

## 七、常见问题（FAQ）

**Q1：为什么 doc/xls/ppt 文件转换失败？**

请确认：
1. 已安装 Microsoft Office。
2. 已安装 `pywin32`：`pip install pywin32`。
3. Office 的 COM 接口可正常调用。

**Q2：为什么 pdf → docx 转换失败？**

请确认已安装 `pdf2docx`：

```bash
pip install pdf2docx
```

**Q3：为什么图片转换后背景变黑了？**

PNG/GIF 等带透明通道的图像转换为 JPG 时，程序会自动将透明区域转为黑色背景。建议先转换为 PNG 保留透明，或转换为支持透明的格式。

**Q4：为什么音频转换失败？**

`pydub` 处理 mp3 等格式通常需要 `ffmpeg`。请安装 ffmpeg 并添加到系统 PATH。

**Q5：转换过程中界面卡死怎么办？**

- 大文件夹扫描或大量文件转换时，界面可能短暂无响应，属于正常现象。
- 若长时间无响应，可尝试减少单次转换文件数量，或关闭程序重新启动。

**Q6：可以转换整个磁盘吗？**

技术上可以添加磁盘根目录，但不建议。扫描整个磁盘会产生大量文件列表，消耗大量内存和时间，且可能包含系统文件。
